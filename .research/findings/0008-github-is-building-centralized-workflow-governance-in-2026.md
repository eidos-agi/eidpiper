## Claim

GitHub's 2026 Actions security roadmap is shifting from distributed per-workflow configuration to centralized policy built on the ruleset framework. The goal: define trust boundaries for workflow execution, secret access, and event triggers without encoding security logic into every workflow file. This is directionally the same problem eidpiper solves, but scoped to security/governance, not release pipeline orchestration or template management.
